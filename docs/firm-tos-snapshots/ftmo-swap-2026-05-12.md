## Triple-rollover convention

FTMO uses the **standard MetaTrader 5 convention**: the **3x** daily swap
is charged at the **Wednesday 22:00 server-time rollover** (so a position
held over the Wed→Thu rollover catches the Sat+Sun+Mon T+2 settlement
period). FTMO's MT5 server runs in **EET/EEST** (UTC+2 in winter, UTC+3
in summer), but the rollover itself is anchored to **22:00 New York time**
(02:00 UTC EDT, 03:00 UTC EST) per the global FX market convention.

| field                       | value                                |
| --------------------------- | ------------------------------------ |
| triple-rollover weekday     | **Wednesday** (Python `weekday()=2`) |
| rollover hour (server time) | 22:00 New York time (US/Eastern)     |
| account types (Phase 0)     | FTMO "FTMO Trader" MT5 (non-swap-free) |
| swap-free / Islamic offered | **Yes** — "Swap-Free Trader" variant; charges a fixed administration fee per night instead of swap. Out of scope for Phase 0 (separate `SwapTable`, deferred). |

### Source-of-record gap

The FTMO public trading-conditions page lists **typical spreads and
commission** explicitly, but routes the per-symbol swap to "see your MT5
terminal → right-click symbol → Properties". The **canonical source for
this table is the MT5 terminal's Symbol Specification dialog**, which is
not reachable from this implementation host (macOS, no MT5 client).

Numbers above are therefore marked **UNCERTAIN** and are seeded from the
**publicly archived FTMO snapshot of mid-2025** (FTMO blog "Trading
costs" page + community-aggregated tables on /r/Forex) so that the unit
tests exercise non-trivial signed rates without inventing a magnitude.
Before any Phase 1 live work, this table must be refreshed from the MT5
terminal on the production VPS and the rows re-flagged CONFIRMED.

### Sign convention

Recorded as published by the broker MT5 server:
* **Positive value** = swap **credit** to the trader (broker pays you).
* **Negative value** = swap **cost** to the trader (broker charges you).

`propfarm.sim.swap.swap_for_position` **inverts** the sign on the way out
so its return value follows the conventional "positive = cost" PnL
convention used by the simulator and rules layer. See module docstring.

### Point-value rationale

For the five-digit FX majors (EURUSD, GBPUSD, USDJPY) and the four-digit
metals (XAUUSD), and the index symbols as quoted by the FTMO data feed,
1 point at 1 lot is approximately worth **1.00 USD** in PnL terms on the
FTMO server group. (USDJPY is in fact ~0.66 USD/point at JPY=150 but the
swap-rate table's *points* are already denominated against the lot
notional — multiplying by 1.0 keeps the existing MT5 "swap × points × lots"
formula unchanged.) The 1.00 conversion factor is therefore the
**MT5-native** form; per-symbol calibration is deferred until calibration
against live FTMO statements in Phase 1.
